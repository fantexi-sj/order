二、接口详细文档
接口1：获取菜品列表
基本信息

项目	内容
接口地址	GET /api/dish/list
是否需要登录	否
功能说明	进入店铺时获取菜品分类及菜品列表
请求参数

参数名	类型	必填	说明
merchant_id	number	是	商家ID
请求示例

PlainText



GET /api/dish/list?merchant_id=1
返回数据格式

JSON



{  "code": 200,  "data": [    {      "id": 1,      "name": "热销推荐",      "dishes": [        {          "id": 1,          "category_id": 1,          "name": "麻辣香锅",          "image": "https://xxx.com/image.jpg",          "price": "15.00",          "unit": "份",          "sales_count": 128,          "status": 1,          "sort_order": 0,          "has_spec": 1        },        {          "id": 2,          "category_id": 1,          "name": "薯条",          "image": "https://xxx.com/image.jpg",          "price": "8.00",          "unit": "份",          "sales_count": 256,          "status": 1,          "sort_order": 1,          "has_spec": 0        }      ]    }  ]}
返回字段说明

字段名	类型	说明
id	number	分类ID
name	string	分类名称
dishes	array	该分类下的菜品列表
dishes[].id	number	菜品ID
dishes[].name	string	菜品名称
dishes[].image	string	菜品图片URL
dishes[].price	string	菜品价格
dishes[].unit	string	单位
dishes[].sales_count	number	销量
dishes[].has_spec	number	是否有规格：0-无 1-有

接口3：添加商品到购物车
基本信息

项目	内容
接口地址	POST /api/cart/add
是否需要登录	是
功能说明	将菜品添加到购物车，支持无规格和有规格两种情况
请求头

PlainText



Authorization: Bearer <token>Content-Type: application/json
请求参数（无规格菜品）

参数名	类型	必填	说明
merchant_id	number	是	商家ID
dish_id	number	是	菜品ID
quantity	number	否	数量，默认1
请求示例（无规格菜品）

JSON



{  "merchant_id": 1,  "dish_id": 2,  "quantity": 1}
请求参数（有规格菜品）

参数名	类型	必填	说明
merchant_id	number	是	商家ID
dish_id	number	是	菜品ID
quantity	number	否	数量，默认1
specs	array	是	选中的规格数组
specs 数组结构

TypeScript



interface SpecItem {  spec_id: number;      // 规格ID  spec_name: string;    // 规格名称  option_id: number;    // 选项ID  option: {    id: number;         // 选项ID    name: string;       // 选项名称    price: number;      // 加价金额  };}
请求示例（有规格菜品）

JSON



{  "merchant_id": 1,  "dish_id": 1,  "quantity": 1,  "specs": [    {      "spec_id": 1,      "spec_name": "辣度",      "option_id": 3,      "option": {        "id": 3,        "name": "中辣",        "price": 0      }    },    {      "spec_id": 2,      "spec_name": "加料",      "option_id": 5,      "option": {        "id": 5,        "name": "加蛋",        "price": 2      }    }  ]}
返回数据格式

JSON



{  "code": 200,  "message": "添加成功",  "data": {    "id": 1,    "quantity": 1,    "unit_price": 17.00,    "total_price": 17.00  }}
返回字段说明

字段名	类型	说明
id	number	购物车记录ID
quantity	number	当前数量
unit_price	number	单价（菜品价格+规格加价）
total_price	number	小计金额（单价×数量）

三、接口汇总表
序号	接口地址	方法	需登录	调用时机
1	/api/dish/list	GET	否	进入店铺
2	/api/spec/dish-specs	GET	否	点击有规格菜品
3	/api/cart/add	POST	是	添加商品到购物车
4	/api/cart/list	GET	是	查看/刷新购物车
5	/api/cart/quantity/:id	PUT	是	修改商品数量
6	/api/cart/remove/:id	DELETE	是	删除购物车商品
7	/api/cart/checkout	GET	是	结算页面
四、金额计算说明
后端自动计算，前端只需展示：

PlainText



单品单价 = 菜品原价 + 规格加价之和单品小计 = 单品单价 × 数量购物车总金额 = 所有单品小计之和
示例：

菜品原价：15元
选择规格：中辣(+0元)、加蛋(+2元)
数量：2份
PlainText



单价 = 15 + 0 + 2 = 17元小计 = 17 × 2 = 34元
